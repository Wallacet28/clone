Instalação e Configuração

Aprenda a instalar o Git em diferentes sistemas operacionais e configure-o para começar a trabalhar com controle de versão.

Instalando o Git no Windows

Acesse https://git-scm.com/download/win;

Faça o download do instalador e execute;

Aceite a licença e clique em “Next”, e siga configurando como desejar¹ e clicando em “Next”;

Finalize clicando em “Install”, e “Finish”.

¹Em "Select Components“, deixe as opções “Git Bash Here” e “Git GUI Here” marcadas.

Instalando o Git no Linux (Ubuntu)

Confira a doc.: https://git-scm.com/download/linux;

Instale a última versão estável do Git:

# add-apt-repository ppa:git-core/ppa
# apt update
# apt install git

Instalando o Git no macOS

Confira a doc.: https://git-scm.com/download/mac;

Instale o Homebrew: https://brew.sh/;

Instale o Git:

$ brew install git

Configurando o Git

$ git config --list

Configurando seu nome de usuário e e-mail (globalmente):

$ git config --global user.name "Nome Sobrenome"
$ git config --global user.email seuemail@email.com

Configurando o nome da Branch Padrão:

$ git config --global init.defaultBranch main

Primeiros Passos com Git e GitHub

Explore como criar repositórios, clonar projetos existentes e realizar operações básicas no Git.

Clonando um Repositório do GitHub

Para clonar um repositório do GitHub no Git, siga os passos abaixo:

  1. Abra o terminal.
  2. Navegue até o diretório onde deseja clonar o repositório.
  3. Use o comando: 
    git clone https://github.com/username/nome-do-repositorio.git
  4. O repositório será clonado para o seu sistema local.

Enviando Atualizações para o Repositório

Depois de fazer alterações no código, siga estes passos para enviar as atualizações:

  1. Use o comando git add para adicionar as alterações ao índice.
  2. Use o comando git commit para confirmar as alterações.
  3. Use o comando git push para enviar as atualizações para o repositório remoto.

Clonando um Repositório do GitHub

Para clonar um repositório do GitHub no Git, siga os passos abaixo:

  1. Abra o terminal.
  2. Navegue até o diretório onde deseja clonar o repositório. Você pode usar o comando cd para navegar entre os diretórios.
  3. Visite o repositório do GitHub que deseja clonar em seu navegador e copie a URL do repositório. A URL será algo como: https://github.com/username/nome-do-repositorio.git
  4. No terminal, use o comando git clone seguido da URL do repositório que você copiou anteriormente. Cole a URL após o comando git clone: 
    git clone https://github.com/username/nome-do-repositorio.git
  5. O Git irá clonar o repositório para o diretório local. Você verá uma mensagem indicando que o clone foi concluído.

Agora você tem uma cópia local do repositório em seu sistema. Você pode começar a trabalhar nele, fazer alterações e enviar atualizações de volta para o repositório no GitHub.

Dicas e Materiais de Apoio

Descubra dicas úteis, recursos de aprendizado e ferramentas relacionadas ao Git e GitHub.

Git Aliases: Adicionando apelidos para os comandos do Git

Exemplo: Criando um alias para o comando status:

$ git config --global alias.st status

Agora podemos utilizar o comando git st para acionar o git status.

Para remover o alias que criamos, utilizamos o seguinte comando:

$ git config --global --unset alias.st

Benefícios do Git

  1. Controle de Versão: O Git permite que você controle todas as versões do seu código.
  2. Colaboração: Facilita a colaboração em equipe, permitindo que vários desenvolvedores trabalhem no mesmo projeto. 
    # Clone um repositório para colaborar
git clone https://github.com/usuario/projeto.git
  3. Rastreamento de Alterações: Registra todas as mudanças feitas no código, quem fez e quando. 
    # Verifique o histórico de commits
git log
  4. Branches: Você pode criar branches para desenvolver recursos separadamente e mesclá-los quando estiverem prontos. 
    # Crie uma nova branch
git branch nome-da-branch

# Mude para a nova branch
git checkout nome-da-branch
  5. Resolução de Conflitos: Ajuda a resolver conflitos quando várias pessoas fazem alterações no mesmo arquivo. 
    # Resolva conflitos durante um merge
git merge branch-de-origem
  6. Backup: Mantém uma cópia de backup do seu código no repositório remoto. 
    # Envie suas alterações para o repositório remoto
git push origin nome-da-branch
  7. Rastreamento de Problemas: Integra-se facilmente com plataformas de rastreamento de problemas, como o GitHub Issues. 
    # Referencie problemas nos commits
git commit -m "Resolve #123: Corrige bug no login"
  8. História de Revisões: Permite revisar facilmente o histórico de revisões do código. 
    # Veja as alterações feitas em um commit específico
git show hash-do-commit
  9. Distribuído: Trabalha de forma distribuída, o que significa que cada desenvolvedor tem uma cópia completa do repositório.
  10. Integração com Ferramentas de CI/CD: Pode ser integrado a ferramentas de integração contínua (CI) e entrega contínua (CD) para automatizar testes e implantações.

Passos para colocar o seu site online e mostra para comunidade

Passo 1: Crie um Repositório no GitHub

Para hospedar seu site no GitHub, você primeiro precisa criar um repositório no GitHub. Siga os passos abaixo:

  1. Acesse o GitHub em https://github.com.
  2. Entre na sua conta ou crie uma nova, se necessário.
  3. Clique no ícone "+" no canto superior direito e escolha "Novo repositório".
  4. Preencha o nome do repositório, escolha a visibilidade e selecione a opção "Incluir um arquivo README".
  5. Clique em "Criar repositório".

Passo 2: Crie uma Página HTML

Para adicionar o conteúdo do seu site, você precisa criar uma página HTML. Siga os passos abaixo:

  1. Crie uma página HTML (por exemplo, "index.html") que contenha o conteúdo do seu site.
  2. Adicione o código HTML ao arquivo.

Passo 3: Faça o Commit e o Push para o GitHub

Para enviar seu site para o GitHub, siga os passos abaixo:

  1. Abra o terminal ou prompt de comando no diretório do seu projeto.
  2. Inicialize um repositório Git, adicione os arquivos e faça o commit.
  3. Conecte seu repositório local ao repositório remoto do GitHub.
  4. Faça o push do código para o GitHub.

Passo 4: Habilite o GitHub Pages

Para tornar seu site público, siga os passos abaixo:

  1. Acesse as configurações do seu repositório no GitHub.
  2. Role para baixo até a seção "GitHub Pages".
  3. No menu suspenso "Branch", escolha a branch onde está o código do seu site.
  4. Clique em "Salvar".
